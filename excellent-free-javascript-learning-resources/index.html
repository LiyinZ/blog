Excellent (free) JavaScript Learning Resources

Flash is dead, JavaScript has won. Nowadays JavaScript is probably the hottest technology to learn, and is rightly so. If you want to do anything web related, you are going to learn JavaScript.

Fortunately for us, we are living in the age of open web and open courses, where countless resources are readily available on the web. At the same time having too many choices could be a problem for beginners. In this blog post, I will try my best to cover some of the best resources for learning JavaScript on the web.

Beginner to Intermediate

CodeCademy (JavaScript Track)

An excellent primer for someone who has no idea what JavaScript is or has no prior programming experience; an interactive tutorial for getting a feel of how coding is like. It could also be a syntax refresher, but its benefits end there. Why? Because too much hand-holding and not enough explanation and programming.

If you just use JavaScript here and there, I suppose this is a fine resource, and you could stop reading here. I would also suggest learning more about jQuery to do add simple functionality on your site.

For someone who is aspired to become a great JavaScript developer, it is a must to move onto something else, and the rest of the article covers what those are.

Eloquent JavaScript

The famed JavaScript book which successfully raised its funding for a new edition, sponsored by big names such as Mozilla and Hack Reactor. It is a really good book for forcing you to think like a programmer, coming up solutions by yourself. It also covers a wide range of topics related to computer science and web from JavaScript data structure to HTTP, with projects to work on after each major part. In my opinion, solving problem on your own and working on a project is the best way to learn.

A fair bit of warning however, some of the examples it provides is quite math intensive. If you are someone who is afraid of math, you might get stuck at some of the beginning chapters.

You Don’t Know JS

A great book on learning the ins and outs of JavaScript. Just read about the philosophy behind the inception of the book (Chapter 3: Into YDKJS). Some key points taken from that section:

  • It is not just some subset of the language that someone called “the good parts,” and not just whatever minimal amount you need to get your job done at work.
  • JS developers seem to stand out from the crowd in the sense of typically not learning very much of the language. This is not a good thing.
  • It challenges you to go beyond your comfort zone and to ask the deeper “why” questions for every single behavior you encounter.

Just by reading the introductory chapter, I have gained much more insight than before where I would just venture a guess of why something happens. If you skipped Eloquent JavaScript, definitely don’t skip this one.

Intermediate to Advanced

I will not go into too much details into these because quite honestly I have not read them yet. But they are on my reading list.

JavaScript Allongé, the “Six” Edition

The “next-step” JavaScript book after the above resources. The six edition mainly uses ES6 (latest JavaScript edition not fully supported in latest browser) as examples. Highly reputable.

JavaScript Design Patter

Design pattern is very necessary if you want to write maintainable, scalable JavaScript application. Must read for professional JavaScript developer.

Others

Other things to learn or read about.

Command Line: lots of wonderful JavaScript packages, add-ons are written in node.js and utilized through NPM, which is mainly used with command line.

Node School: another reason to learn command line. Node school provides a wide range of JavaScript tutorial through the command line, and they are all free, from absolute beginner to advanced topic. I would recommend it more than codecademy but it requires having node.js installed and knowing command line, so I place it here for your consideration.

JavaScript styleguide: written by Airbnb with 23,300 stars on GitHub as of writing. Following a certain style makes your code cleaner and much more maintainable.

Conclusion

There are many more out there, but these are the resources I highly recommend and am going through personally.
