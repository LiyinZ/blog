How to start using Hexo to blog - part 2

In the first part of How to start using Hexo to blog, you learned setting up your blog locally. Now, it’s time to bring it online! Specifically, we are going to deploy it to GitHub. Why you may ask? Because essentially GitHub provides free hosting if you know how, and it’s actually quite simple.

GitHub Pages

Assuming you already have a GitHub account, there are basically 2 ways for creating GitHub Pages.

Method 1 - github.io

This method provides a way to create a page for the base github.io URL. For example, if your user name is LiyinZ (actually that is mine), the URL for your GitHub profile would be https://github.com/LiyinZ, and naturally, http://LiyinZ.github.io belongs to you. The same goes for organizations. Say you have an organization called Facebook, your repositories would be hosted at https://github.com/Facebook, and the URL http://Facebook.github.io belongs to you as well.

In order to create a page for the base xxxx.github.io URL, all you need to do is create a repository named xxxx.github.io URL, and git push some contents with index.html inside the root directory. For example, here’s mine.

That’s it! Easy enough? The only downside with this method is that it can only be created once per user or organization, but if you think about it, it makes total sense- one page per one github.io base URL.

This method is officially called User & Organization Pages, you can check out the official doc here.

Method 2 - /gh-pages

The 2nd way is as simple. Do you know you can have a landing page for each of your projects? Provided that you include an index.html in your project root directory. Yes, this method is called Project Pages.

All you have to do, within your project, create a branch called gh-pages. You can directly do it on your project page like this:

Once the branch is created, with index.html inside your root directory, your page is live. Simply navigate to username.github.io/your-project-name to view it. For example, my assignment project https://github.com/gc-assignments/comp2081-a03‘s URL is http://gc-assignments.github.io/comp2081-a03/.

Deploy Hexo Blog to GitHub Pages

Now that you understand the basics of GitHub pages, lets take our blogs online!

Hexo actually provides a super easy way to deploy to GitHub, all you have to do
is to type the command hexo deploy and it automatically does everything for
you!

Conclusion

As you can see, Hexo is really easy to use with its built in commands and configs. In my opinion, it is better than the most popular static site generator Jekyll. For instance, you only need the hexo new <post-name> command, instead of creating a file following a strict yyyy-mm-dd-post-name naming scheme to create a new post.

For more static site generator choices, take a look at the wonderful site staticgen.com.
